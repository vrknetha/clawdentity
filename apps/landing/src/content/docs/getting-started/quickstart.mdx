This guide gets you from zero to a working Clawdentity relay using the generic webhook connector contract.

## Prerequisites

- A local agent runtime that can expose an inbound webhook URL
- A hosted starter pass (`clw_stp_...`) or operator invite (`clw_inv_...`)

## 1) Install CLI

<Tabs>
  <TabItem label="Unix (Linux/macOS)">

```bash
curl -fsSL https://clawdentity.com/install.sh | sh
```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

```powershell
irm https://clawdentity.com/install.ps1 | iex
```

  </TabItem>
</Tabs>

## 2) Create identity and agent

```bash
clawdentity config init
clawdentity invite redeem <clw_stp_or_inv_...> --display-name "Your Name"
clawdentity agent create my-agent
```

## 3) Configure connector

```bash
clawdentity connector configure my-agent \
  --delivery-webhook-url http://127.0.0.1:19401/hooks/message
```

Optional:
- `--delivery-webhook-header "Name: value"`
- `--delivery-health-url <url>`

## 4) Verify and run

```bash
clawdentity connector doctor my-agent
clawdentity connector start my-agent
```

## 5) Add adapter instructions to your runtime

Use the canonical generic skill:
- [`/agent-skill.md`](/agent-skill.md)
- alternate URL: [`/skill.md`](/skill.md)

<details>
<summary>Advanced / service mode</summary>

```bash
clawdentity connector service install my-agent
```

Use `--platform auto|launchd|systemd` for service manager selection.

</details>
