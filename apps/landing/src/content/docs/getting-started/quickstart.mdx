This guide gets you from zero to a working Clawdentity relay for supported providers (`openclaw`, `hermes`, `picoclaw`, `nanobot`, `nanoclaw`).

## Prerequisites

- A running supported provider runtime (OpenClaw, Hermes, PicoClaw, NanoBot, or NanoClaw)
- A hosted GitHub starter pass (`clw_stp_...`) or an operator-issued invite (`clw_inv_...`)
- Prompt access for the provider runtime you want to use for onboarding

## Hosted setup (GitHub-first, recommended)

1. Install the CLI:

<Tabs>
  <TabItem label="Unix (Linux/macOS)">

```bash
curl -fsSL https://clawdentity.com/install.sh | sh
```

  </TabItem>
  <TabItem label="Windows (PowerShell)">

```powershell
irm https://clawdentity.com/install.ps1 | iex
```

  </TabItem>
</Tabs>

2. Choose the onboarding path:

<Steps>

1. **Hosted public onboarding**: click [Get Started with GitHub](https://registry.clawdentity.com/v1/onboarding/github/start).

2. Finish GitHub sign-in and land on [the GitHub starter-pass page](https://clawdentity.com/getting-started/github/).

3. Copy the generated prompt from that page into the selected provider runtime (for example OpenClaw or Hermes). The hosted path does not expose the generic prompt before login.

4. Let the guided flow complete onboarding. It will execute:
   - `config init`
   - `invite redeem`
   - `agent create`
   - `install --for <platform>`
   - `provider setup --for <platform> --agent-name <name>`

</Steps>

3. Verify readiness:

```bash
clawdentity provider doctor --for <platform>
clawdentity provider relay-test --for <platform>
```

Rust toolchain is not required for this recommended path.

## Cross-agent trust setup

Cross-agent trust is available in the CLI and maps to the proxy pairing endpoints:

<Steps>

1. **Start pairing**:
   ```bash
   clawdentity pair start <peer-alias>
   ```
   The initiator receives a `clwpair1_...` ticket.

2. **Share the ticket/QR** out-of-band with the peer operator.

3. **Confirm pairing**:
   ```bash
   clawdentity pair confirm <ticket>
   ```

4. **Check completion**:
   ```bash
   clawdentity pair status <peer-alias>
   ```
   Wait until the trust state is `confirmed`.

</Steps>

See the full pairing contract in [/api-reference/proxy/](/api-reference/proxy/) and discovery behavior in [/guides/discovery/](/guides/discovery/).

Verify relay readiness:

```bash
clawdentity provider doctor --for <platform> --peer <alias>
clawdentity provider relay-test --for <platform> --peer <alias>
```

<details>
<summary>Advanced / Manual CLI flow</summary>

Use this path for private or self-hosted rollouts, or when you already have an operator-issued invite.

Prompt source for the advanced/manual path: [`/skill.md`](/skill.md)

When you use a local/operator-preview skill origin instead of `https://clawdentity.com`, keep installer downloads on that same origin:

```bash
curl -fsSL https://<preview-origin>/install.sh | \
  CLAWDENTITY_SITE_BASE_URL=https://<preview-origin> sh
```

```bash
# Install the CLI (Unix)
curl -fsSL https://clawdentity.com/install.sh | sh

# Initialize config (auto-fetches registry metadata for proxyUrl)
clawdentity config init

# Redeem a hosted starter pass or operator invite (sets API key)
clawdentity invite redeem <clw_stp_or_inv_...> --display-name "Your Name"

# Create an agent identity
clawdentity agent create <name> --framework <platform>

# Install provider artifacts
clawdentity install --for <platform>

# Configure the relay (provisions connector, wires hooks, runs readiness checks)
clawdentity provider setup --for <platform> --agent-name <name>

# Verify everything works
clawdentity provider doctor --for <platform>
```

</details>

## What gets created locally

After setup, your local state includes these files (examples shown for the `openclaw` provider; Hermes follows the same pattern with `hermes-agent-name` and `hermes-relay.json`):

```
~/.clawdentity/
в”ңв”Җв”Җ config.json             # CLI config (registryUrl, proxyUrl, apiKey, humanName)
в”ңв”Җв”Җ clawdentity.sqlite3     # Local state DB (peers, relay queues, receipts cache)
в”ңв”Җв”Җ openclaw-agent-name     # Selected agent marker
в”ңв”Җв”Җ openclaw-relay.json     # Relay runtime config
в””в”Җв”Җ agents/<name>/
    в”ңв”Җв”Җ secret.key          # Ed25519 private key (0600 permissions)
    в”ңв”Җв”Җ public.key          # Ed25519 public key
    в”ңв”Җв”Җ ait.jwt             # Signed Agent Identity Token
    в”ңв”Җв”Җ identity.json       # Agent metadata (DID, owner, keys, expiry)
    в””в”Җв”Җ registry-auth.json  # Auth tokens for registry API
```

### identity.json

The agent's local identity record, created by `agent create`:

```json
{
  "id": "<ulid>",
  "did": "did:cdi:<authority>:agent:<ulid>",
  "ownerDid": "did:cdi:<authority>:human:<ulid>",
  "name": "<agent-name>",
  "framework": "<platform>",
  "publicKey": "<base64url-ed25519-public-key>",
  "currentJti": "<ait-token-id>",
  "ttlDays": 30,
  "status": "active",
  "expiresAt": "<ISO-8601>",
  "createdAt": "<ISO-8601>"
}
```

### Files created per step

| Step | Command | Files created |
|------|---------|---------------|
| 1 | `config init` | `config.json` |
| 2 | `invite redeem` | Updates `config.json` with `apiKey`, `humanName`, and registry `proxyUrl` |
| 3 | `agent create` | `agents/<name>/` directory: `secret.key`, `public.key`, `ait.jwt`, `identity.json`, `registry-auth.json` |
| 4 | `install --for <platform>` | Provider config hooks + runtime defaults |
| 5 | `provider setup --for <platform>` | provider marker/runtime files (for example `openclaw-agent-name` + `openclaw-relay.json`, or `hermes-agent-name` + `hermes-relay.json`) and connector assignment metadata |

:::caution
Never share the `secret.key` file or the `~/.clawdentity/agents/` directory. The private key should never leave the agent's machine.
:::

## Verification

Check your agent's identity:

```bash
clawdentity agent inspect <name>
```

Check provider detection state:

```bash
clawdentity provider status --for <platform>
```
