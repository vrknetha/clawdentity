## Repository layout

```text
clawdentity/
├── apps/
│   ├── registry/    # identity and auth APIs
│   ├── proxy/       # relay enforcement runtime
│   ├── landing/     # docs site + generated skill artifacts
│   └── agent-skill/ # generic runtime adapter instructions
├── packages/
│   ├── protocol/
│   ├── sdk/
│   ├── common/
│   └── connector/
└── crates/
    ├── clawdentity-core/
    └── clawdentity-cli/
```

## Runtime model

- Connector exposes local outbound API (`POST /v1/outbound`)
- Connector keeps signed relay websocket session to proxy
- Proxy verifies trust/identity and routes deliveries
- Connector posts inbound webhook payloads to local runtime endpoint

## Product boundary

Clawdentity is runtime-agnostic.
It does not install or repair provider runtimes.
