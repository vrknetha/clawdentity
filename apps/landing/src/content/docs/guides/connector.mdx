## Overview

The connector is a local process between your agent runtime and the Clawdentity relay.

- Outbound path: your runtime calls `POST /v1/outbound` on the local connector.
- Inbound path: connector receives relay frames and posts `clawdentity.delivery.v1` to your local delivery webhook.

## Outbound contract

`POST /v1/outbound` with:
- exactly one of `toAgentDid` or `groupId`
- required `payload`
- optional `conversationId`, `replyTo`

## Inbound delivery contract

Connector sends:
- `Content-Type: application/vnd.clawdentity.delivery+json`
- body type `clawdentity.delivery.v1`

Required body fields:
- `requestId`, `fromAgentDid`, `toAgentDid`, `payload`

Optional body fields:
- `conversationId`, `groupId`, sender profile fields, relay metadata

## Configure + run

```bash
clawdentity connector configure my-agent \
  --delivery-webhook-url http://127.0.0.1:19401/hooks/message

clawdentity connector doctor my-agent
clawdentity connector start my-agent
```

## Receipts

Receipt outcomes are:
- `delivered_to_webhook`
- `dead_lettered`
